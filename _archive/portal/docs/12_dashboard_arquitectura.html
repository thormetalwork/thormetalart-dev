1. VISIÓN GENERAL DEL SISTEMA

1.1 Objetivo del dashboard

El dashboard es una aplicación web privada (solo para Thor Metal Art y su consultor) que centraliza todas las métricas digitales del negocio en una sola pantalla. Reemplaza la necesidad de entrar a cuatro plataformas distintas (Google Analytics, Google Business Profile, Instagram, Excel) para entender el estado del negocio.

META

Karel debe poder abrir el dashboard en 30 segundos y saber exactamente: cuántos leads tiene este mes, qué está generando tráfico, cuántas reseñas tiene, y qué acciones necesita tomar hoy. Sin necesidad de conocimiento técnico.

1.3 Principios de diseño del sistema

  • Karel primero: la UI está diseñada para alguien sin conocimiento técnico. Si algo no se entiende en 5 segundos, se rediseña.
  • Datos reales, no estimaciones: cada número en el dashboard viene de una API verificada o de una entrada manual explícita.
  • Bajo mantenimiento: el sistema debe funcionar sin intervención técnica mensual. Las APIs se actualizan solas; el token de Instagram requiere renovación semestral.
  • Móvil-primero: Karel probablemente revisará el dashboard desde el teléfono. Responsive design no es opcional.
  • Costo operativo mínimo: el stack elegido debe tener hosting gratuito o cerca de $0/mes para el estado actual del negocio.

3.4 Autenticación y seguridad

El dashboard es privado — solo Karel y el consultor pueden acceder. Se usa NextAuth.js con Google OAuth como proveedor de identidad.

  • Login: Google OAuth 2.0 — el usuario presiona 'Continuar con Google' y se autentica con su cuenta @gmail o @thormetalart.com
  • Whitelist de emails: solo los emails autorizados (Karel + consultor) pueden ingresar. Cualquier otro Google account recibe acceso denegado.
  • Sesión: JWT con expiración de 30 días. Karel no necesita hacer login cada vez.
  • Credenciales de APIs: NUNCA en el frontend. Todas las API keys y tokens de OAuth se guardan como variables de entorno en Vercel (NEXT_PUBLIC_ solo para datos no sensibles).
  • HTTPS obligatorio: Vercel provee SSL automático. Todo el tráfico es HTTPS.
  • Token de Instagram: el long-lived token (60 días) se guarda encriptado en Supabase. La app lo refresca automáticamente cada 55 días usando un cron job.

6. GUÍA DE SETUP — PASO A PASO

TIEMPO ESTIMADO

Setup completo: 6-10 horas para un desarrollador con experiencia en Next.js. La mayor parte del tiempo es la configuración de cada API (OAuth flows, permisos, credenciales). El código en sí es relativamente sencillo.

6.1 Fase de setup — Infraestructura

  1. Crear repositorio GitHub: thormetalart-dashboard (privado)
  2. Inicializar Next.js 14: npx create-next-app@latest --typescript --tailwind --app
  3. Crear proyecto en Vercel y conectar al repositorio GitHub
  4. Crear proyecto en Supabase: Free tier — región: US East (N. Virginia)
  5. Crear las 3 tablas en Supabase usando el SQL editor (leads, metrics_cache, alerts)
  6. Configurar variables de entorno en Vercel (ver sección 6.4)

6.2 Fase de setup — Google APIs (GA4 + GBP)

  1. Crear proyecto en Google Cloud Console: 'ThorMetalArt Dashboard'
  2. Habilitar APIs: Analytics Data API · My Business Performance API · My Business Account Management API
  3. Crear credenciales OAuth 2.0: Application type = Web application · Redirect URIs = https://dashboard.thormetalart.com/api/auth/callback/google
  4. Crear Service Account para GA4 (server-to-server, no requiere interacción del usuario)
  5. Agregar el Service Account como Viewer en la propiedad GA4 de Thor Metal Art
  6. Descargar el JSON key del Service Account y guardarlo como variable de entorno en Vercel
  7. Para GBP: Karel debe autorizar la app con su cuenta de Google que tiene acceso al GBP

6.3 Fase de setup — Instagram Graph API

  1. Crear app en Meta for Developers: developers.facebook.com → tipo 'Business'
  2. Agregar producto: Instagram Graph API
  3. Verificar que @thormetalart sea Business Account (no Creator ni Personal)
  4. Conectar la cuenta de Instagram Business a una Facebook Page de Thor Metal Art
  5. Obtener User Access Token con scopes: instagram_basic, instagram_manage_insights, pages_read_engagement
  6. Convertir a Long-Lived Token (60 días): GET https://graph.facebook.com/v18.0/oauth/access_token?grant_type=fb_exchange_token&...
  7. Guardar el long-lived token en Supabase (tabla: config) encriptado con AES-256
  8. Configurar cron job para renovación automática cada 55 días

7.2 Criterios de aceptación — Checklist de QA antes del go-live

  • Login funciona con la cuenta Google de Karel y la del consultor
  • Cuentas no autorizadas reciben acceso denegado (whitelist validada)
  • GA4: sesiones, canales, top páginas y conversiones del mes corriente se muestran correctamente
  • GBP: impresiones, acciones y las últimas 5 reseñas se muestran con fecha correcta
  • Instagram: seguidores, alcance y los últimos 10 posts se cargan correctamente
  • Leads: Karel puede crear, editar y actualizar estado de un lead en menos de 30 segundos
  • Alertas: al menos 3 reglas de alerta funcionan correctamente con datos de prueba
  • PageSpeed Insights: mobile ≥ 70 · desktop ≥ 85
  • Sin errores de consola en producción (Chrome DevTools)
  • Cron jobs ejecutan sin errores y actualizan la DB (verificar en Vercel logs)
  • Karel prueba el dashboard desde su teléfono y lo aprueba
